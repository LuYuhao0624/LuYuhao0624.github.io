Deploy Introduction on GitHub with Hugo (Windows)

Oct 28, 2020 | 3 minutes read
Share this: Twitter Facebook Pinterest Google+

After 2 days of trial and error, I finally deployed the example site of Introduction theme on my GitHub homepage. Follow this tutorial to avoid all troublesome warnings and errors on Windows. To give you a bit of background, I am totally new to foreground stuff, like html, js and so on, so I believe every one can handle it.

Basically, you can follow the tutorial on Hugo docs and Introduction. To rebuild the example site, just copy all the files in <SITE ROOT>/themes/introduction/exampleSite into <SITE ROOT>, override all duplications. And then you can preview your website by hugo server in the site root directory. BTW, as long as the server is not terminated, you can preview all your changes immediately, very impressive.

But now, if you visit your website, you might find that some icons do not show up and turn into some annoying rectangles. Also, the local time at the bottom of the page is probably not loaded. If you do not get these problems, then congrats, customize your website now.

For the first problem, go to edit the config.toml file in your site root. If the value of baseURL starts with http://, then change it into https://. This will solve your problem, rebuild, push to GitHub and you will see it succeeds (it may take a while for these changes to take effect on your website).

The second problem is in the Issues of Introduction GitHub, but that thread does not have a solution. You will receive an error like “Failed to find a valid digest in the ‘integrity’ attribute” if you open the console in your website (ctrl+shift+J for Chrome). This is because the end of line is changed from LF to CRLF in Windows when using Git (if you have it configured). The way to solve this is to reset all the CRLF to LF. For users not very familiar with Git, before git add . in public directory, you can configure Git by git config --global core.autocrlf false and add, commit and push. If you have already added the files, update with git add . -u. After pushing the changes to remote, you can set it back to your original settings. Now go to your website to see that everything works fine!

For Windows users, I also write a .bat file to automate the upload procedure.

git config --global core.crlf false

hugo -D

cd public

git add .

git commit -m "updated"

git push origin master

git config --global core.crlf true

save the content in a file, e.g. upload.bat, in the site root and double click on it to push all changes to remote.
